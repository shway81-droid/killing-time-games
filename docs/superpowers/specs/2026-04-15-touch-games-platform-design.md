# Touch Games Platform — Design Spec

## Overview

학교 교실의 대형 터치스크린에서 초등학생(5~9세)이 쉽게 플레이할 수 있는 웹 기반 게임 플랫폼. 혼자 또는 여러 명이 함께 플레이 가능. 게임은 지속적으로 추가될 예정이며, 첫 번째 게임은 i-Spy 스타일의 숨은 그림 찾기.

## Constraints

- **환경**: 학교 교실, 대형 터치스크린, 항상 온라인
- **대상**: 초등학생 5~9세
- **플레이어**: 1명 또는 다수 (게임에 따라 다름)
- **데이터 저장**: 없음 — 매번 새로 시작
- **콘텐츠**: placeholder 그래픽으로 시작, 이후 실제 이미지로 교체
- **기술 스택**: 순수 HTML/CSS/JavaScript (Vanilla) — 빌드 도구 없음

## Project Structure

```
touch-games/
├── index.html          ← 메인 런처 (게임 선택 화면)
├── css/
│   └── common.css      ← 공통 스타일 (터치 최적화, 폰트 등)
├── js/
│   └── common.js       ← 공통 유틸 (터치 이벤트, 화면 전환 등)
├── games/
│   ├── i-spy/
│   │   ├── index.html  ← 숨은 그림 찾기 게임
│   │   ├── style.css
│   │   ├── game.js
│   │   └── assets/     ← 이미지, placeholder 등
│   ├── game-2/         ← 이후 추가될 게임
│   └── ...
└── assets/
    └── icons/          ← 런처에서 쓸 게임 아이콘 등
```

- 각 게임은 `games/` 아래에 독립 폴더로 존재
- 게임 추가 시 폴더만 만들고 런처에 등록하면 완료
- 빌드 도구 없음 — 파일을 웹서버에 올리면 바로 동작

## Launcher (메인 화면)

- 큰 카드 형태의 게임 목록 (터치하기 쉽게 최소 200x200px 이상)
- 게임 이름 + 아이콘/썸네일 + 간단한 설명
- 터치하면 해당 게임의 `index.html`로 이동
- 각 게임에서 "뒤로가기" 버튼으로 런처로 복귀
- 화면 방향: 가로 모드 고정 (대형 터치스크린 기본 방향)

## Touch Interaction Design

- **탭(터치)**: 오브젝트 선택, 버튼 클릭
- **드래그/스와이프**: 큰 그림에서 화면 이동 (숨은 그림 찾기 등)
- **핀치 줌**: 확대/축소 (그림을 자세히 볼 때)
- 마우스 이벤트도 동시 지원 (개발/테스트 편의)
- 더블탭 줌, 우클릭 메뉴 등 브라우저 기본 동작 비활성화
- 터치 영역 최소 48px 이상 (어린이 손가락 고려)
- 정답 터치 시 시각적 피드백 (색상 변화, 애니메이션)

## Game 1: I-Spy (숨은 그림 찾기)

### Game Flow

1. 테마 선택 화면 (예: 교실, 운동장, 숲 등)
2. 큰 배경 일러스트가 표시됨
3. 화면 상단/하단에 "찾아야 할 아이템 목록" 표시
4. 그림을 드래그/줌하며 탐색
5. 아이템을 찾으면 탭 → 정답 시 체크 표시 + 효과음/애니메이션
6. 모두 찾으면 축하 화면 → 다른 테마 선택 또는 런처로 복귀

### Placeholder Strategy

- 배경: 컬러 블록 + 간단한 도형으로 구성된 장면
- 숨길 아이템: 기본 SVG 아이콘 (별, 하트, 동물 등)
- 나중에 실제 일러스트로 교체 가능한 구조 (이미지 파일만 바꾸면 됨)

### Difficulty

- 쉬움: 아이템 3개
- 보통: 아이템 5개
- 어려움: 아이템 8개

## Out of Scope

- 사용자 인증/로그인
- 점수 저장/랭킹 시스템
- 오프라인 모드
- 실제 일러스트/이미지 제작 (placeholder로 시작)
- 사운드/효과음 (추후 추가 가능)
